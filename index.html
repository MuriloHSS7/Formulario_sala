Perguntas

  1. Na tag <input> com atributo type="text", para que serve o atributo "placeholder"?

    2. Resposta: O atributo "placeholder" em uma tag <input> com o atributo "type" definido como "text" é usado para
    fornecer uma dica ou um texto de orientação para os usuários sobre o tipo de entrada que é esperado no campo de
    texto. Ele é exibido no campo de texto antes de o usuário digitar qualquer coisa e desaparece assim que o usuário
    começa a digitar..

  2. Na tag <input> com atributo type="radio" ou type="checkbox", para que serve o atributo "checked"?

    3. Resposta: O atributo "checked" em uma tag <input> com o atributo "type" definido como "radio" ou "checkbox"
    é usado para determinar se o elemento de entrada (input) deve ser inicialmente marcado (checked) quando a página é
    carregada no navegador. Ele é um atributo booleano, o que significa que, se estiver presente, o elemento será marcado,
    e se estiver ausente, o elemento não será marcado.

  3. Na tag <input> para que serve o atributo "value"?

    4. Resposta: O atributo "value" na tag HTML <input> é usado para definir o valor inicial ou o valor atual de um
    elemento de entrada (input) em um formulário. Ele especifica o valor que será exibido no campo de entrada quando a
    página for carregada ou quando o campo de entrada for criado.

  4. Que atributo deve ser utilizado para agrupar vários botões de rádio em um formulário de modo que apenas um deles fique marcado de cada vez?

    5. Resposta: O atributo que deve ser utilizado para agrupar vários botões de rádio em um formulário de modo que apenas
    um deles fique marcado de cada vez é o atributo "name". Todos os botões de rádio que pertencem ao mesmo grupo devem
    ter o mesmo valor para o atributo "name". Isso permite que o navegador saiba que esses botões fazem parte de um grupo
    e que apenas um deles pode ser selecionado de cada vez.

  5. Para que serve a tag <label> Para que serve o atributo "for" nessa tag?

    6. Resposta: A tag <label> em HTML é usada para associar um rótulo descritivo a um elemento de formulário, como uma
    caixa de seleção (input), campo de texto, botão de opção (radio button) ou qualquer outro elemento de entrada. A principal
    finalidade do elemento <label> é melhorar a acessibilidade da página da web, tornando mais fácil para os usuários
    entenderem qual entrada de formulário está relacionada a que descrição. O atributo "for" na tag HTML <label> é usado para vincular
    um rótulo a um elemento de entrada (como um campo deentrada de texto, caixa de seleção ou botão de opção) em um formulário.

  6. Na tag <option> para que serve o atributo "selected"?

    7. Resposta: atributo "selected" na tag <option> é usado em elementos de lista suspensa (dropdown) em formulários HTML
    para indicar qual opção deve ser exibida como selecionada por padrão quando a lista é inicialmente carregada ou exibida ao
    usuário. Isso permitde que você defina qual opção deve ser a opção predefinida quando a página é carregada, em vez de deixar
    a primeira opção como selecionada aumaticamente

  7. Nas tags de formulário para que serve o atributo "required"?

    8. Resposta: O atributo "required" em tags de formulário HTML é usado para indicar que um determinado campo (como um campo de
    entrada de texto, caixa de seleção, caixa de verificação, etc.) é obrigatório e deve ser preenchido pelo usuário antes que
    o formulário possa ser enviado. Quando um campo é marcado como "required", o navegador da web verifica automaticamente se o
    usuário preencheu esse campo antes de permitir o envio do formulário.

  8. Para que serve a tag <textarea>?

    9. Resposta: A tag <textarea> é usada em HTML para criar uma área de texto expansível em um formulário da web. Ela permite que
    os usuários insiram texto livremente, que pode ser usado para coletar informações, fazer comentários, escrever mensagens
    longas e outras interações onde o usuário precisa inserir texto. A tag <textarea> é especialmente útil quando você deseja
    permitir que os usuários escrevam mais do que algumas palavras, tornando-a uma alternativa prática aos campos de entrada
    de texto de uma única linha.

  9. Na tag <form> para que serve o atributo "action"?

    10. Resposta: O atributo action na tag <form> em HTML é usado para especificar o URL (Uniform Resource Locator) para onde os
    dados do formulário serão enviados quando o formulário for submetido. Em outras palavras, ele define o destino ou o local
    onde as informações do formulário serão processadas.

  10. Na tag <form> para que serve o atributo "method"?

    11. Resposta: O atributo "method" na tag <form> em HTML serve para especificar o método HTTP que será usado para enviar os dados
    do formulário para o servidor web quando o formulário é submetido. Este atributo determina como os dados do formulário serão
    transmitidos ao servidor.

  11. Para que o valor digitado de uma senha não apareça na barra de endereços do navegador, qual deve ser o valor do atributo "method" na tag <form>?

    12. Resposta: Nesse caso, o elemento

    deve ter o atributo method definido com o valor post, para que a senha não
    seja mostrada na linha de endereço.

  12. Qual valor deve ser informado no atributo "method" na tag <form> em um formulário que será usado para fazer upload de arquivos?

    13. Resposta: Para criar um formulário que permita o envio de arquivos (como imagens, documentos, etc.) para o servidor,
    você deve definir o valor do atributo "method" na tag <form> como "POST". A carga de arquivos geralmente requer o uso
    do método "POST" porque os dados binários dos arquivos não podem ser adequadamente codificados em uma URL, como
    aconteceria com o método "GET". Portanto, para ações de envio de arquivos, o atributo "method" deve ser definido como br "POST".

  13. O que as tags <fieldset> e <legend> produzem?

    14. Resposta: As tags HTML <fieldset> e <legend> são usadas em conjunto para criar grupos de elementos em um formulário e
    fornecer uma legenda ou título para esse grupo. Elas são utilizadas para melhorar a acessibilidade e a organização de
    formulários, principalmente quando você tem campos relacionados que devem ser agrupados visualmente.
